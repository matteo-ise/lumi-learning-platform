LUMI · Problem

The Challenge

The Learning Gap Problem

  • 1 in 4 primary school students (grades 1–4) falls significantly behind peers without personalized support
  • Private tutoring costs €30–60/hour — unaffordable for most families
  • Classroom teachers cannot provide 1-on-1 instruction at scale (avg. 25 students per class)
  • Existing apps deliver static exercises with no adaptive feedback or Socratic guidance
  • Motivation gap: children disengage quickly without gamification and instant positive reinforcement

LUMI · Solution

The Solution

LUMI's Approach

  • Socratic Method: LUMI never gives the answer directly — it guides students step-by-step through targeted questions
  • AI-Powered Personalization: Each session is shaped by the student's grade, learning style, and curriculum (Bundesland)
  • 3-Step Tutoring Method: Foundation → Deepening → Practice — mirrors expert human tutors
  • Multimodal Interaction: Voice input/output, image recognition (homework photos), Apple Pencil support
  • Gamification Loop: LUMI Blast math game + daily streaks keep motivation high

LUMI · Model Selection

Why Gemini 2.0 Flash?

Model Selection Rationale

  • 25× Cheaper Cost: Gemini 2.0 Flash vs. GPT-4o — enables free-tier sustainable deployment
  • Vision Multimodal: Native image understanding for homework photo analysis (no OCR pipeline needed)
  • Quality Accuracy: Excellent for grades 1–4 curriculum — clear, age-appropriate explanations out of the box
  • Speed Latency: <2 s median response — critical for maintaining child attention span
  • No RAG Architecture: Full curriculum context fits in Flash's 1M token context — eliminates vector DB complexity

LUMI · Prompt Engineering

AI Design

Prompt Engineering: Meta-Prompts

  • Wizard Profile: Onboarding collects name, grade (1–4), Bundesland, avatar, learning style

  • Curriculum Context: Grade-specific Markdown lesson plans loaded from backend/knowledge/ into prompt

LUMI · Feature 1

Core Pedagogy

3-Step Tutoring Method

  • Structured Learning Arc: Every topic follows Foundation → Deepening → Practice — no shortcuts
  • HotKey Buttons: "Got it ✓", "Example please", "Harder" drive flow without free-text friction
  • Adaptive Pace: Student responses determine whether LUMI revisits, advances, or introduces a new angle
  • Socratic Enforcement: Backend prompt rejects any request to reveal answers; reroutes to guiding questions

① Foundation

Core concept introduced with age-appropriate analogy. LUMI checks prior knowledge first.

② Deepening

Guided Socratic dialogue — questions narrow understanding gaps iteratively.

③ Practice

Student solves a problem independently; LUMI gives structured feedback, not solutions.

LUMI · Feature 2

Input / Output Modes

Multimodal Interaction

  • Voice Input (STT): Web Speech API — children speak questions; no keyboard required for grades 1–2
  • Voice Output (TTS): Browser speech synthesis reads LUMI's responses aloud for pre-readers
  • Image Recognition: Students photograph homework worksheets; Gemini Vision analyzes and responds inline
  • Apple Pencil / Stylus: iPad-optimized graph-paper chat background; draw directly in the chat area
  • No Extra APIs: All modalities powered by Gemini Vision + Web Platform APIs — zero third-party cost

LUMI · Feature 3

Motivation Systems

Gamification

  • LUMI Blast: HTML5 Canvas math-shooter game — students answer grade-level equations to fire at targets
  • Adaptive Difficulty: Game questions are pulled from the same curriculum context as the tutor sessions
  • Streak System: Daily login + learning session tracked; streak counter shown prominently on dashboard
  • Avatars & Badges: Chosen during onboarding wizard; unlockable rewards tied to streak milestones
  • Intrinsic Motivation: Gamification layers are optional — core tutoring works independently of game progress

LUMI · Feature 4

Adaptive Learning

Personalization Engine

  • Onboarding Wizard: 5-step flow captures name, avatar, grade (1–4), German Bundesland, learning style preference
  • Context-Aware Responses: Gemini receives full student profile on every request — responses match vocabulary level exactly
  • Curriculum Alignment: Bundesland selection loads the matching state syllabus into the knowledge context
  • Learning Style Modes: Visual / Auditory / Reading-Writing profiles alter explanation style in the system prompt
  • Profile Persistence: Profile stored in Firebase / SQLite; consistent across devices and sessions

LUMI · Deployment

Production & Roadmap

Deployment & Scalability

Current Setup

  • Frontend: Vercel CDN — global edge, zero cold starts
  • Backend: Railway / Render — single FastAPI instance (stateless)
  • DB: Neon (serverless PostgreSQL) — auto-scales to zero
  • Auth: Firebase — Google-managed, GDPR/DSGVO-compliant

Scale-Out Roadmap

  • Horizontal: Stateless API → Kubernetes / Cloud Run replicas
  • Caching: Redis layer for repeated curriculum context blocks
  • Streaming: Gemini streaming responses for <300 ms TTFB
  • Analytics: BigQuery pipeline for learning progress dashboards
